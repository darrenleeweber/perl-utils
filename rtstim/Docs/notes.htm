Notes


  • If one is collecting responses from a subject, then a file named response.txt will be written to the associated paradigm directory. This file contains the buttons pressed and the time they were pressed relative to the scheduling of the response event.
  • The utility load_rt_stim_module is used to install the module rt_stim.o. It needs to get the absolute path of rt_stim.o from environment variable STIM_RELEASE_PATH. So if this environment variable name is changed, the program load_rt_stim_module.c has to be changed as well.
  • When building a new paradigm directory, sound_list.txt file is a must even you will not be using sound file in your experiment. A blank sound_list.txt can be used in this case.
  • The example image files from 6.bmp to 20.bmp are just the repeat copies from file 1.bmp to file 5.bmp which is just an image displaying a number [1-5].
  • Example for respond.txt can refer to file paradigms/resp/respond.txt.bak .
  • image_list.txt and sound_list.txt DO NOT ALLOW the blank line. This will be changed later.
  • User has to have super user's priviledge in order to compile the RTStim. Because installing module need the superuser's priviledge.
