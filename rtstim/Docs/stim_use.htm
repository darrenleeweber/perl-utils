Using RTStim

There are two modes in which to use RTStim: Static mode and Dynamic mode. Most of you will probably want to use static mode but you should be aware of the capabilities of each mode .  Static mode can be used when the experimenter wants to create a fixed sequence of events. Static mode requires no programming by the user and a new Sequnce Builder GUI is avavilable in RTStim 2.0. Dynamic mode is used when the sequence of events is not fixed. The use of dynamic mode requires programming by the user. Please have a look at the RTStim API if you would like to use dynamic mode.

The following is a list of the type of events available in RTStim:

Visual Stimuli

  • Displays .bmp images
  • Image resolution of 800x600 or 1024x768
  • Image depth 8-bit, 16-bit or 24-bit
  • BitBlitting (the horizontal and vertical dimensions must be a multiple of 4)
  • Synchronization with the vertical retrace
  • Short movies
Auditory Stimuli
  • Playback of .wav files (uses 44.1kHz sampling rate)
  • 24-bit depth audio
Input/Output
  • Hardware interrupt synchronization with external hardware using the motherboard parallel port.
  • Three 8-bit ports for IO on the PCI DIO-24 card Response-pad interface is hardware interrupt driven on the PCI DIO-24 card.

1. Running the Examples

To run the static mode examples run "RTCentralApp" then in the GUI enter:
Stimulus Directory: /home/rtstim/rtstim_release_2.0/paradigms/xxxx/
User Application: /home/rtstim/rtstim_release_2.0/stim/stim
where xxxx is a paradigm directory  (see Building a Sequence) choosen from the list below. Press the "GO" button to start the sequence of scheduled events. If the application is not found you should verify that the PATH environment variable was set correctly during the installation. This can be checked by doing "which RTCentralApp". If the PATH is set correctly, the absolute path of the RTCentralApp program will be displayed.
  • bitblt/: display bitblt images on a specified background images with different positions.
  • demo/: display some full screen images.
  • movie/: display bounch of full screen images with given frame rate, just like playing a movie.
  • sync/: example of sync event.
  • resp/: example of respond event.
  • stereo/: example of playing sound event.

    • If you are having any difficulties getting RTStim to run please send us the RTStim log file (see installation).

    2. Building a Sequence

      To build a static mode sequence there are to methods: (1) Use the GUI sequence builder or (2) Manually. Regardless of the method used to build a sequence, the first step in building a working sequence is the creation of a paradigm directory. You can name the paradigm directory whatever you like but each paradigm directory must contain the following three files:
       
    • sequence.txt:  the list of scheduled events, the event times and event properties (built by Sequence Builder GUI or manually).
    • image_list.txt:  a list of absolute paths to the .bmp files used by the sequence.
    • sound_list.txt:  a list of absolute paths to the .wav files used by the sequence.

      •  

       

      The sequence.txt file uses a syntax of "label=xxx" where label may designate the event  type or some specific parameter associated with that event and xxx designates the value given to that label.  For a complete table of events and associated parameter labels follow this link. Please have a look at the sequence.txt files for each of the examples above. This will give you a quick introduction to the proper structure of a sequence file. The Sequence Builder GUI is convenient in that it gives you choices for parameter labels associated with a choosen event type.

    3. Running a Sequence

    To run your paradigm's sequence file start the "RTCentralApp" then in the appropriate text GUIs enter:
    Stimulus Directory: /home/rtstim/rtstim_release_2.0/paradigms/xxxx/
    User Application: /home/rtstim/rtstim_release_2.0/stim/stim
    where xxxx is the paradigm directory  for your paradigm. Press the "GO" button to start the sequence of scheduled events. Whe using static mode the User Application is always that given above . When using dynamic mode the User Application will be the user-space program you have created using the RTStim API, and the paradigm directory will contain the image_list.txt and sound_list.txt files but not the sequence.txt file.

    4. RTStim Utilities

  • sim_sync is a user-space program (located in the Utils/ directory) that will allow you to simulate a synchronization with external hardware.
  • sim_keypad is a user-space program (located in the Utils/ directory) that will allow you to simulate a response pad interaction with RTStim.
  • For appropriate electrical connections between the RTStim machine's PCI DIO-24 port and the parallel port of the Linux machine (usually a laptop) running the simulation programs please refer to the file named "connect".
  • a log file utility for examining any error messages produced as the RTStim module is loaded (see installation).
